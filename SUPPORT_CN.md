# 寻求帮助 · Support

## 文档

大多数问题都能在项目文档里找到答案：

- [README_CN.md](README_CN.md) — 总览、特性、快速上手
- [docs/USER_GUIDE_CN.md](docs/USER_GUIDE_CN.md) — 完整使用指南（最推荐）
- [docs/QUICKSTART_CN.md](docs/QUICKSTART_CN.md) — 5 分钟上手
- [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) — 系统架构

## 社区

- **GitHub Discussions** — 提问、想法、随便聊：
  <https://github.com/badhope/MCP-HUB/discussions>
- **GitHub Issues** — 报告 bug、提 feature request：
  <https://github.com/badhope/MCP-HUB/issues>

## 怎样提问题

| 问题类型 | 去哪里问 |
|------------------|--------------|
| "某个功能怎么用？" | Discussions，没人答再发 Issue |
| "我发现了一个 bug" | Issues（用 bug 模板） |
| "我有个想法 / feature request" | Discussions → Ideas 分类 |
| "我想加一个 MCP 服务器" | 用 `server_submission_CN` 模板开 Issue |
| "我发现一个安全问题" | **见 [SECURITY_CN.md](SECURITY_CN.md)，不要在公开 issue 里写** |
| "我想贡献代码" | 见 [CONTRIBUTING.md](CONTRIBUTING.md) |

## 响应时间预期

这是开源项目，**尽力而为**，**没有 SLA**。典型响应时间：

- 严重安全问题：24 小时内
- Bug 报告：1 周内
- Feature request / 提问：1 个月内，可能根本没空

如果你需要 SLA，请 fork 仓库自己维护。

## 商业支持

暂不提供商业支持。如果你需要紧急加功能或修 bug，请写一个清晰的 issue，并考虑赞助开发。
