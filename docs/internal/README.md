that are useful for maintainers but **not intended for end users**.

| `COMPLETENESS_SCORING_GUIDE.md` | Design notes for the server-completeness scoring algorithm |
| `DOMESTIC_COMPANIES_REPORT.md` | Historical survey of Chinese-company MCP coverage |
| `FULL_SYSTEM_GUIDE.md` | Pre-rewrite full-system walkthrough (kept for context) |
| `IMPROVEMENT_PLAN.md` | Earlier round of planned improvements |
| `NOTABLE_PROJECTS_GUIDE.md` | List of upstream projects tracked |
| `PROJECT_STRUCTURE_GUIDE.md` | Historical project-structure walkthrough |
| `REPOSITORY_INTRO.md` | Original repo-introduction draft |
| `SERVER_CATALOG.md` | Original hand-curated server catalog |

If you are an end user, you probably want the [top-level README](../../README.md)
or [docs/API.md](../../docs/API.md) instead.
